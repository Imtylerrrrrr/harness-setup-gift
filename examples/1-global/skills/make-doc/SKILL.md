# 문서 만들기

어떤 프로젝트에서든 쓸 수 있는 범용 스킬입니다.
(범용이니까 글로벌 `~/.claude/skills/` 에 둡니다.)

## 순서

1. **독자와 목적을 한 줄로 확인한다.**
   - 누가 읽는가? 읽고 나서 무엇을 하면 성공인가?
   - 불분명하면 문서를 쓰기 전에 물어본다.
2. **목차(뼈대)를 먼저 제안하고 승인받는다.**
   - 뼈대 없이 바로 긴 글을 쓰지 않는다.
3. **승인된 뼈대에만 살을 붙인다.**
   - 없는 내용을 지어내지 않는다. 모르면 모른다고 표시한다.
4. **마지막 점검.**
   - 맞춤법, 링크가 실제로 열리는지, 코드 블록이 실제로 실행되는지 확인한다.

## 규칙

- 문단은 짧게. 제목은 내용과 일치하게.
- 좋은 예시 하나가 설명 백 줄보다 낫다.
- 독자가 비전문가라면 전문용어 옆에 쉬운 말을 붙인다.
